Implement a clean female livestock lifecycle system in miFinca for the Animals list page and individual Animal profile page.

GOAL:
Keep the UX extremely simple for farmers while allowing the system to track stage duration, countdowns, alerts, and automatic stage transitions over time.

STACK / STYLE:
- Match the current miFinca visual style exactly
- Keep UI minimal and clean
- Do not clutter cards with too much info
- Prioritize readability and simple farmer-first language
- Build this in a reusable way so it can support cattle first, and later other species if needed

========================================
1. LIFECYCLE MODEL
========================================

Create a female reproductive lifecycle with these 5 stages:

1. growing
2. can_breed
3. in_heat
4. pregnant
5. nursing

This should be the main lifecycle shown in the product.

Definitions:
- growing = female animal is not yet eligible to breed
- can_breed = eligible to breed now
- in_heat = currently in heat for a short breeding window
- pregnant = confirmed pregnant
- nursing = has delivered and is currently nursing offspring

Important:
- This lifecycle is only for female animals
- Male animals should not show this lifecycle
- Females too young to breed should show growing
- Adult females should move through the loop:
  growing -> can_breed -> in_heat -> pregnant -> nursing -> can_breed

========================================
2. DATA MODEL CHANGES

Add fields to the animal model. Use naming that fits the current codebase, but include this logic:

Core lifecycle fields:

Pregnancy tracking:

Nursing tracking:
- nursingStartedAt: datetime | null

Optional linkage:

Alert fields / derived helpers:

========================================
3. DEFAULT BUSINESS RULES
========================================

Implement configurable defaults for cattle females:

- minimum breeding age = 730 days
- minimum breeding weight = 400 kg
- heat duration = 3 days
- pregnancy duration = 283 days
- pregnancy health check = 30 days after pregnancy starts
- nursing/weaning duration = 270 days

Use constants/config so these can be changed later by species or farm.

Example config object:
LIFECYCLE_CONFIG = {
  cattle: {
    female: {
      minimumBreedingAgeDays: 730,
      minimumBreedingWeightKg: 400,
      heatDurationDays: 3,
      pregnancyDurationDays: 283,
      pregnancyHealthCheckDays: 30,
      nursingDurationDays: 270
    }
  }
}

========================================
4. DERIVED LOGIC
========================================

Build reusable helper functions for lifecycle status.

Create helper functions like:

- getAnimalAgeDays(animal)
- isEligibleByAge(animal)
- isEligibleByWeight(animal)
- isEligibleToBreed(animal)
- getLifecycleStage(animal, now)
- getLifecycleCountdown(animal, now)
- getNextLifecycleEvent(animal, now)
- getLifecycleAlerts(animal, now)

Rules:

A) growing
A female animal is in growing if:
- not pregnant
- not nursing
- not in heat
- and not eligible to breed yet

Eligibility to breed is true when:
- ageDays >= minimumBreedingAgeDays
OR currentWeightKg >= minimumBreedingWeightKg

Use OR here because the user specifically wants:
“If calf weighs 400kg or is 2 years old it can breed”

B) can_breed
A female animal is can_breed if:
- not pregnant
- not nursing
- not in heat
- and eligible to breed

C) in_heat
A female animal is in_heat if:
- heatStartedAt exists
- now < heatEndsAt
- not pregnant

When heatStartedAt is set:
- auto set heatEndsAt = heatStartedAt + heatDurationDays
- lifecycleStage = in_heat
- lifecycleStageStartedAt = heatStartedAt
- lifecycleStageEndsAt = heatEndsAt

Auto transition:
- when now >= heatEndsAt
- if animal was not marked pregnant or bred, auto move to can_breed

D) pregnant
A female animal is pregnant if:
- pregnancyStartedAt exists
- now < expectedDeliveryAt
- not marked delivered yet

When pregnancy starts:
- expectedDeliveryAt = pregnancyStartedAt + pregnancyDurationDays
- pregnancyCheckDueAt = pregnancyStartedAt + pregnancyHealthCheckDays
- lifecycleStage = pregnant
- lifecycleStageStartedAt = pregnancyStartedAt
- lifecycleStageEndsAt = expectedDeliveryAt

Auto transition:
- when now >= expectedDeliveryAt
- auto move to nursing
- set nursingStartedAt = expectedDeliveryAt
- set nursingEndsAt = nursingStartedAt + nursingDurationDays
- set weaningDueAt = nursingEndsAt

Important:
Also allow a manual “Mark Delivered” action that moves to nursing immediately, in case real delivery date is known and different from estimate.

E) nursing
A female animal is nursing if:
- nursingStartedAt exists
- now < nursingEndsAt

When nursing starts:
- lifecycleStage = nursing
- lifecycleStageStartedAt = nursingStartedAt
- lifecycleStageEndsAt = nursingEndsAt

Auto transition:
- when now >= nursingEndsAt
- auto move to can_breed IF animal is still eligible to breed
- else move to growing

========================================
5. AUTOMATIC STAGE TRANSITIONS
========================================

Implement lifecycle auto-transition logic.

Need 2 levels:

1. Derived front-end logic:
- UI should always display the correct stage based on timestamps and current time even before database cleanup jobs run

2. Persistent background updater:
- create a service/job that scans animals and writes stage transitions when thresholds are crossed
- this can run periodically (ex: every hour or daily depending on architecture)
- do not rely only on cron for UI correctness

Create something like:
- syncAnimalLifecycle(animal, now)
- syncAllAnimalLifecycles(now)

This should:
- determine expected stage based on timestamps + weight + age
- compare with stored lifecycleStage
- if different, persist the new stage
- update lifecycleStageStartedAt / EndsAt
- create lifecycle history event
- update nextLifecycleEventAt / nextLifecycleEventType

Important:
Never auto move male animals into this lifecycle.

========================================
6. ALERT LOGIC
========================================

Create simple farmer-facing alerts.

Alerts to support:
- in_heat_ending_soon
- pregnancy_check_due
- delivery_soon
- ready_to_wean
- eligible_to_breed

Rules:
- In heat ending soon: show when less than 1 day remains in heat
- Pregnancy check due: show at or after 30 days pregnant until completed/dismissed
- Delivery soon: show when <= 30 days until expected delivery
- Ready to wean: show when nursing has reached 270 days or less than 7 days remain
- Eligible to breed: show when growing animal becomes eligible

Return alerts in a clean structure:
[
  {
    type: 'pregnancy_check_due',
    label: 'Pregnancy check due',
    dueAt: ...
    severity: 'warning'
  }
]

========================================
9. ACTION BEHAVIOR
========================================

Implement the actions.

Mark In Heat:
- set heatStartedAt = now or chosen date
- set heatEndsAt based on config
- move lifecycle to in_heat
- create lifecycle history event

Mark Pregnant:
- allow choosing pregnancy start date
- clear heat if needed
- set pregnancyStartedAt
- set expectedDeliveryAt
- set pregnancyCheckDueAt
- move lifecycle to pregnant
- create lifecycle history event

Record Check:
- mark pregnancy check as completed
- store completion date and optional notes

Mark Delivered:
- allow actual delivery date
- end pregnancy
- start nursing from delivery date
- calculate nursingEndsAt and weaningDueAt
- move lifecycle to nursing
- create lifecycle history event

Wean Calf:
- end nursing
- move animal to can_breed if eligible, otherwise growing
- create lifecycle history event

End Heat:
- clear heat dates or mark heat completed
- move back to can_breed if eligible, otherwise growing

========================================
12. EDGE CASES
========================================

Handle these safely:

- animal is male -> no female lifecycle
- animal has no birth date -> use weight only when available
- animal has no weight -> use age only when available
- animal has neither age nor weight -> cannot determine breeding eligibility, show neutral fallback
- pregnancy manually recorded before heat -> allow it
- actual delivery date differs from expected date -> use actual delivery date and recalculate nursing
- overdue pregnancy or overdue nursing should still display correctly and raise alert state instead of breaking
- if lifecycleAutoManaged is false, preserve manual stage unless user explicitly changes it, but still show warnings if dates imply inconsistency
